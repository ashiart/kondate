# 献立メーカー

「今日の夕食が思いつかない」を解消する、献立のきっかけ作りWebツール。
ボタンを押すと主菜・副菜・汁物を1品ずつ提案します。

## UI文言ポリシー

- 短い
- ひと目で意味が分かる
- 説明しすぎない

例：「ぜんぶ変える（再抽選）」ではなく「ぜんぶ変える」

## 使い方（開発中の動作確認）

`index.html` をダブルクリックしてブラウザで開くだけ。
サーバー・ビルド・インストールは不要です。

## ファイル構成

| ファイル | 役割 |
|----------|------|
| `index.html` | 画面の骨組み（3タブ＋カード） |
| `css/style.css` | 全スタイル |
| `js/data.js` | ★初期料理データ。料理を増やすときはここだけ編集 |
| `js/storage.js` | LocalStorage読み書き（M2で作成予定） |
| `js/app.js` | 抽選ロジック・画面描画・イベント処理 |

## 料理の追加方法

1. `js/data.js` を開く
2. 既存の料理1件をコピーして配列に追記し、内容を書き換える
3. ファイル冒頭の「★次に使うID」を新しい料理の `id` に使い、★の数字を1つ進める

### 料理ID運用ルール（絶対に守る）

- IDは「d + 3桁連番」（d001, d002, ...）で、**一度使ったら永久不変**
- 料理を削除しても、そのIDは**欠番にして再利用しない**
  - 例）d003 を削除しても、次に追加する料理は d004
- お気に入い・非表示・履歴はIDで料理を参照しているため、
  IDを変更・再利用すると利用者の設定が壊れる
